# Noise Input Control
## Definition
Using non-speech mouth sounds such as a pop or a hiss as low-latency discrete input events, standing in for clicks or key presses.
## Why it matters here
Some hands-free actions (click, select) need to be faster and less fatiguing than a spoken word; noise triggers give a near-instant, voice-sparing input channel.

- [[command-grammar]] — noise events extend the command vocabulary with non-verbal triggers
