**Abstract (paraphrased from publisher summary — not verbatim):**
Eye-tracking technology can substitute for conventional input devices (mouse, keyboard) to manipulate web content, but it is error-prone for pointing and for executing pointed objects: small executable targets cause pointer-execution errors, leading to unintended activation and difficulty pointing. The authors propose an interface that addresses these two pointer-execution error types for people with disabled upper limbs controlling a GUI. The interface (1) dynamically magnifies the tracked/target executable objects and (2) executes the object pointed at by eye-tracking using a voice command. They implement it as a Chrome extension embedded in the browser.

**Contributions / claims (as captured):**
- Separates the Midas-touch / unintended-activation problem into two pointer-execution error types (pointing error + execution error).
- Dynamic magnification of the gaze-targeted object to reduce pointing error on small targets.
- Voice command as the explicit confirmation/execute trigger to avoid dwell-based unintended activation.
- Delivered as a Chrome browser extension for web GUI operation.

**Method (as captured):** Gaze positions the cursor; the candidate target is dynamically magnified; a voice command executes ("clicks") the pointed object. Implemented as a Chrome extension. A usability evaluation was conducted (details behind paywall).
