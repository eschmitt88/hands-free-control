- 2026-07-01 synth.py noise-sizing fixed (coordinator review). Replaced the analytic small-angle `_sigma_px` with `_calibrate_std_px`, which bisects the per-axis pixel std so the produced mean angular error (exact `geometry.angular_error_deg` over the validation targets) matches `--inject-deg` — accounting for the 2D radial-vs-per-axis factor and off-centre targets. Two further fixes: (a) synth features are built on the PIXEL scale, since `Ridge(alpha=1.0)` on normalized features (XtX ~ alpha) halved the identity-map coefficients and floored recovery at ~6.7 deg; (b) calibration is generated noise-free so the fitted map is the exact identity (avoids errors-in-variables attenuation that biased recovery down, growing with angle), with validation/test carrying the injected noise replicated `_REPS_PER_TARGET=25x` to converge the mean. Selfcheck bounds tightened to [0.8*, 1.25*]inject. Recovered: inject 2.0->1.946, 4.0->3.882, 8.0->7.900 deg — all PASS.
- 2026-07-01 RidgeCalibrator made scale-invariant (coordinator review): wrapped in a sklearn Pipeline `StandardScaler() -> Ridge(alpha, fit_intercept=True)`. The feature-scale shrinkage found in synth also biases the REAL calibration — raw gaze_features (len-6, ~0.01-0.1) and headpose_features (len-5, degrees) have heterogeneous small scales, so a fixed alpha on un-standardized features shrinks the large coefficients needed to reach pixel scale unevenly. Standardizing makes alpha meaningful and independent of each feature's natural scale (correct default for a ~9-point, 6-11-feature per-user calibration). .fit/.predict interface and load_session_jsonl/stack_features signatures unchanged, so analyze.py/synth.py untouched. Selfcheck re-run: inject 2.0->2.123, 4.0->3.693, 8.0->7.253 deg — all PASS within [0.8*,1.25*]. All seven files py_compile clean.
